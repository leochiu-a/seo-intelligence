Add a `useEffect` with an empty dependency array `[]` that runs exactly once on mount (per D-04):

1. Read `localStorage.getItem('seo-planner-graph')`.
2. If null/undefined, do nothing — the empty `initialNodes`/`initialEdges` already apply.
3. If present, `JSON.parse` it. Wrap in try/catch — on parse failure, do nothing (corrupt data = start fresh).
4. For restored nodes: re-add the `onUpdate: onNodeDataUpdate` callback to each node's data (since it was stripped during save). Also set `type: 'urlNode'` if missing.
5. For restored edges: re-add `onLinkCountChange: onEdgeLinkCountChange` to each edge's data. Also set `type: 'linkCountEdge'` and `markerEnd: { type: MarkerType.ArrowClosed, color: '#9CA3AF' }` if missing.
6. Call `setNodes(restoredNodes)` and `setEdges(restoredEdges)`.
7. Set `isRestoring.current = false` after calling setNodes/setEdges.

**Dependency array:** Use `[]` (empty array). This is a one-time mount effect. The `onNodeDataUpdate` and `onEdgeLinkCountChange` callbacks referenced inside the effect are stable `useCallback` hooks whose identities do not change, so closing over them at mount time is safe. Do NOT add them to the dependency array — the effect must run only once.

**Placement:** This effect must be defined BEFORE the save effect in the component body so React processes them in order.
