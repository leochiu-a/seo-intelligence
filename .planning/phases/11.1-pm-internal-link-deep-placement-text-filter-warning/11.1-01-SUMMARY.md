## HealthStatus Interface (Plan 02 Contract)

```typescript
export interface HealthStatus {
  links: 'ok' | 'warn';            // never 'na' — outbound always computable
  depth: 'ok' | 'warn' | 'na';     // 'na' when depthMap empty or nodeId missing
  tags:  'ok' | 'warn';            // never 'na'
}

export function getHealthStatus(
  node: Node<UrlNodeData>,
  depthMap: Map<string, number>,
  outboundMap: Map<string, number>,
): HealthStatus;

export function hasAnyWarning(status: HealthStatus): boolean;
```

## Decisions Made

- `getHealthStatus` placed immediately before `classifyScoreTier` in `graph-utils.ts` — groups Phase 11.1 exports together near the outbound threshold constant they reuse
- `depthMap.size === 0 || !depthMap.has(node.id)` covers both no-root and node-not-in-map cases for the `'na'` branch
- `hasAnyWarning` uses `=== 'warn'` (not `!== 'ok'`) so future metric values like `'na'` are not accidentally treated as warnings
