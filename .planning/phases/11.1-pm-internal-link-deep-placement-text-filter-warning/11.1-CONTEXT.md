# Phase 11.1: PM Health Check Panel - Context

**Gathered:** 2026-04-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 11.1 adds a PM Health Check view inside the existing right sidebar — accessible via a [Score | Health] tab toggle at the sidebar top. The Health view shows every node as a compact list row with 3 warning badge columns (Links, Depth, Tags), plus a "Show warnings only" toggle to surface only dangerous pages. No canvas interaction from the health panel (read-only).

**In scope:** Score/Health tab toggle in sidebar, 3-metric health row component, "Show warnings only" filter toggle.

**Out of scope:** Per-metric granular filtering, canvas node selection from health rows, Placement as a direct health metric (it feeds PageRank score, not a standalone warning), anchor text diversity warnings (Phase 12).

</domain>

<decisions>
## Implementation Decisions

### Panel Entry Point
- **D-01:** Health Check lives **inside the existing right sidebar** (ScoreSidebar slot). Two tabs at the sidebar top: `[Score] [Health]`. Clicking a tab switches the sidebar content area. No layout change to the rest of the app.
- **D-02:** Tab state is **ephemeral** (local component state, not persisted). Reloading the page returns to Score view.

### Metric Definitions
- **D-03:** The health panel tracks **3 metrics** per node:
  1. **Links** — outbound link count vs `OUTBOUND_WARNING_THRESHOLD` (150). Warning = `outboundCount > 150`. Value from `outboundMap` (already computed in App.tsx).
  2. **Depth** — crawl depth vs threshold. Warning = `depth > 3` or `depth === Infinity` (unreachable). Value from `depthMap`. No root set → depth unknown → no badge rendered (same as current sidebar behavior).
  3. **Tags** — topical cluster tags. Warning = `node.data.tags` is empty/undefined. OK = node has ≥1 tag.
- **D-04:** **Placement is NOT a health metric.** It is a parameter in the PageRank score calculation, not a direct page-level warning. The phase title listed it descriptively, not as a separate health dimension.
- **D-05:** Orphan and unreachable nodes appear in the health list with Depth = warning. No separate section (unlike the Score sidebar's orphan/unreachable grouping).

### Layout / Visual
- **D-06:** Each row renders: URL template (truncated) + 3 icon badges. Badge states: `warning` (red icon, visible) / `ok` (muted/green icon, visible) / `n/a` (hidden, for Depth when no root is set).
- **D-07:** Rows are **read-only** — clicking a row does nothing. Health panel is a diagnostic read-out, not a navigation tool.
- **D-08:** Icon choices (Claude's discretion, keep lucide-react vocabulary tight):
  - Links: `Link2` or `ArrowUpRight` — red when over-linked
  - Depth: `Layers` — red when deep/unreachable
  - Tags: `Tag` — red when untagged
- **D-09:** Row sort order: warnings-first (rows with any warning float to top), then alphabetical by URL template. This ensures the PM sees dangerous pages without needing to enable the toggle first.

### Filter
- **D-10:** A single "Show warnings only" toggle (checkbox + label) sits at the top of the Health view, above the list. When checked, rows with all-OK status are hidden.
- **D-11:** Toggle state is local (no persistence). Defaults to unchecked (show all).

### Claude's Discretion
- Exact badge layout within each row (e.g., right-aligned badges vs inline after URL)
- Tab styling (pill tabs vs underline tabs) — use whichever fits the existing sidebar header aesthetic
- Whether to add a summary line at the top of the Health view: e.g. "3 of 12 pages have warnings" (probably yes — useful at a glance)
- TDD: pure helper `getHealthStatus(node, depthMap, outboundMap): { links: 'ok'|'warn', depth: 'ok'|'warn'|'na', tags: 'ok'|'warn' }` is the natural unit-testable boundary

</decisions>

<specifics>
## Specific Ideas

- Summary line at top of Health view: "3 / 12 pages have warnings" — quick PM scan without reading every row.
- Warning-first sort: rows with ≥1 warning appear above all-OK rows. Within each group, sort alphabetically by URL template. Pure, stable, testable.
- Badge layout suggestion: URL template left-truncated (flex-1 overflow-hidden) + right-aligned badges in a `flex-shrink-0` row. Badges are icon-only (no text) to fit the narrow sidebar. Tooltip on hover showing the actual value (e.g. "Depth: 5") is a nice-to-have.

</specifics>
