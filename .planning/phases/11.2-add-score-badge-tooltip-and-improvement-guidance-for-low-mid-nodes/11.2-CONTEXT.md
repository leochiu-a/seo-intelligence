# Phase 11.2: Score Badge Tooltip & Improvement Guidance - Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Add hover tooltips to the Low/Mid/High score tier badges on canvas nodes. Each tooltip explains what the tier means (PageRank rank in the overall distribution) and provides a one-line actionable hint for how to improve the score. Canvas nodes only — sidebar score list is out of scope for this phase.

**In scope:** Tooltip on Low, Mid, and High badges in `UrlNode.tsx`, triggered on hover of the badge span itself. Tooltip content = tier explanation + improvement action.

**Out of scope:** Sidebar score list tooltips (next phase), changing the badge visual design, changing the scoring algorithm, Neutral badge tooltip (Neutral means all pages scored equally — no action needed).

</domain>

<decisions>
## Implementation Decisions

### Tooltip content
- **D-01:** Every tier badge gets a tooltip (Low, Mid, High — not Neutral).
- **D-02:** Tooltip content structure: **explanation + how to fix**.
  - Low: explain PageRank fell in the bottom third of all nodes + suggest adding more internal links pointing to this page from higher-scoring pages.
  - Mid: explain PageRank is in the middle third + suggest similar action to elevate it.
  - High: explain PageRank is in the top third + confirm it's well-connected (no action needed, just affirmation).
- **D-03:** Tone should be PM-friendly (non-technical). Avoid "PageRank algorithm" jargon — say "link equity" or "internal link strength" instead.

### Trigger
- **D-04:** Hover on the badge `<span>` itself — no extra `?` icon. The badge becomes the tooltip trigger directly.
- **D-05:** Use the existing `Tooltip`, `TooltipTrigger`, `TooltipContent` components from `packages/web/src/components/ui/tooltip.tsx`. `TooltipProvider` already wraps the app in `App.tsx` (no changes to App.tsx needed).

### Scope
- **D-06:** Canvas nodes only (`UrlNode.tsx`). Sidebar score list deferred.
- **D-07:** Neutral badge is NOT modified — Neutral means all nodes scored equally (no meaningful gradient to act on), so a tooltip adds no value.

### Claude's Discretion
- Exact tooltip copy (wording) — keep it under ~2 sentences, PM-friendly, actionable for Low/Mid, affirming for High.
- Whether to use `side="top"` or `side="bottom"` for the tooltip positioner (pick whichever avoids canvas edge clipping — `side="top"` is probably safer since nodes can be near the bottom).
- Whether to add `sideOffset` tuning to avoid overlap with the badge itself.

</decisions>

<specifics>
## Specific Ideas

- Extending `TONE_MAP` with a `tooltip` field per tier is the cleanest approach — keeps all tier display data in one place and avoids scattered string literals.
- Example tooltip text direction (exact copy is Claude's discretion):
  - Low: "Low link equity — this page receives fewer internal links than 2/3 of your pages. Point more links here from high-scoring pages to improve it."
  - Mid: "Mid link equity — this page is in the middle third. Adding more internal links from high-scoring pages can push it higher."
  - High: "High link equity — this page is well-connected and receives strong internal link support. Keep it linked from key pages."
- Neutral badge has no tooltip (D-07).

</specifics>
