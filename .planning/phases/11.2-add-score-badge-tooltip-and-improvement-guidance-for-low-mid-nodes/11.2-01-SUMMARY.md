# Phase 11.2 Plan 01: Add Score Badge Tooltip Summary

Wrapped the Low / Mid / High tier badges on canvas nodes with Base UI Tooltip components. Hovering a badge now shows a PM-friendly explanation of the tier plus a one-line action to improve it (or an affirmation for High). Five vitest tests cover the tooltip trigger presence per tier plus absence for neutral/undefined.

## Tooltip Copy Shipped

**High:**
> "High link equity — this page is in the top third of internal link strength across your graph. It is well-connected; keep it linked from your most important pages."

**Mid:**
> "Mid link equity — this page sits in the middle third of internal link strength. Add more internal links from high-scoring pages to push it higher."

**Low:**
> "Low link equity — this page receives less internal link strength than two-thirds of your pages. Point more internal links here from high-scoring pages to improve it."

**Neutral:** No tooltip (D-07 — Neutral means all nodes scored equally; no actionable gradient).

## Implementation Notes

- Added `tooltip: string` field to `TONE_MAP` (all 4 tiers; `neutral` gets `""` to satisfy exhaustive Record)
- Added `import { Tooltip, TooltipTrigger, TooltipContent } from "@/components/ui/tooltip"` to UrlNode.tsx
- Replaced bare `<span>` badge with `<Tooltip><TooltipTrigger render={<span />} data-testid="badge-tooltip-trigger" data-tier={tier}>{tone.badgeLabel}</TooltipTrigger><TooltipContent>{tone.tooltip}</TooltipContent></Tooltip>`
- `TooltipProvider` NOT added — the existing app-level provider in App.tsx is reused (D-05)
- No `?` icon added — the badge span itself is the trigger (D-04)
- Global badge and Root badge are unchanged (D-06)

## Confirmations

- **D-07 (Neutral excluded):** Confirmed. The existing `{tier !== "neutral" && (...)}` gate covers the entire Tooltip wrapper — no tooltip is rendered for neutral tier or undefined scoreTier.
