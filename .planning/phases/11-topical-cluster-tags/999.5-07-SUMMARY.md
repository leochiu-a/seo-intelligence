# Phase 999.5 Plan 07: FilterPanel By cluster + AND-combine + Dim Fix Summary

FilterPanel cluster filter, placement x cluster AND-combine, and stripe-persistent dim shipped together as one cohesive plan.

## What Was Built

### Task 1: collectClusterGroups (TDD)

New `ClusterGroup` interface and `collectClusterGroups(nodes)` utility added to `graph-utils.ts` directly after `collectPlacementGroups`. Key differences from placement groups: includes BOTH global and non-global nodes, groups by `tags` array rather than placements.

Behavior: dedupes duplicate tags within a single node, skips empty/whitespace-only strings, sorts alphabetically by tagName. Full TDD RED/GREEN cycle with 5 test cases.

### Task 2: UrlNode DOM refactor + App.tsx styledNodes

**UrlNode restructure:**
- Added cluster stripe: `data-testid="cluster-stripe"` as an absolutely-positioned sibling of the content wrapper — always full opacity
- Added content wrapper: `data-testid="card-content"` wrapping edit button, badges, title, subtitle — receives `style={{ opacity: data.isDimmed ? 0.2 : 1 }}`
- `isDimmed?: boolean` added to `UrlNodeExtendedData` interface
- Cluster stripe renders up to 3 equal bands; overflow count shown in subtitle
- Tag chips rendered in subtitle row with color dot + tag name

**App.tsx styledNodes:**
- Replaced `node.style.opacity` approach with `data.isDimmed` propagation
- Plain object-literal spread `{ ...node.data, isDimmed }` mirrors existing enrichment pattern (no type assertion)
- Drop-shadow filter removed entirely

### Task 3: FilterPanel + App.tsx AND-combine

**FilterPanel.tsx rewrite:**
- Section header renamed from "Placement Filters" to "By placement"
- New "By cluster" section added below placement section
- Cluster section conditionally rendered — hidden entirely when `clusterGroups.length === 0`
- Filter key format: `cluster:{tagName}` (placement keys unchanged: `placement-name:{name}`)

**App.tsx highlightedNodeIds:**
- Keys split by prefix into `placementKeys` and `clusterKeys`
- Per-dimension Set unions computed: `placementMatches`, `clusterMatches`
- AND-combination: when both dimensions active, intersect the two Sets
- When only one dimension active, return that dimension's Set

## FilterPanel Section Order

Placement first, cluster second. This order reflects the existing placement section being primary (placement filters were shipped in Phase 7); cluster is the new dimension added in this wave.

## AND-combine Intersection Logic As Shipped

```typescript
if (placementKeys.length > 0 && clusterKeys.length > 0) {
  return new Set([...placementMatches].filter((id) => clusterMatches.has(id)));
}
if (placementKeys.length > 0) return placementMatches;
return clusterMatches;
```

Within each dimension: OR logic (multiple selections expand the set). Across dimensions: AND logic (intersection narrows the result).

## Drop-shadow Filter Decision

**Dropped.** The drop-shadow was applied via `node.style` which is set by React Flow on the `react-flow__node` wrapper — the same element whose opacity was being controlled. With the new `data.isDimmed` approach, `node.style` is not touched at all, making drop-shadow inapplicable. The dim effect is purely via inline `opacity` on the `card-content` wrapper inside UrlNode.
