## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test cases 1 and 6 could not pass as originally specified**

- **Found during:** Task 2 (GREEN phase — tests still failed after implementation)
- **Issue:** The PLAN spec described Test 1 as "two-node A→B, same cluster → score(B) > control". But when node A has only one outbound edge to B, the `CLUSTER_BONUS_FACTOR` cancels in the ratio: `effectiveLinkCount / totalWeightedOut_A = (1.5 × lc) / (1.5 × lc × pageCount_B) = same as without bonus`. The bonus only manifests when A has edges with mixed cluster states (some same-cluster, some cross-cluster), creating an asymmetry in the numerator that the unified denominator does not cancel.
- **Fix applied:** Test 1 was rewritten as "A links to both B (same cluster) and C (cross cluster) with equal linkCount — B gets more equity than C". Test 6 was rewritten as "two global nodes G1 (same cluster as A) and G2 (different cluster) — G1 scores higher than G2". Both correctly verify the bonus effect without changing the implementation math.
- **Files modified:** `packages/web/src/lib/graph-utils.test.ts`
- **Commit:** 53da86b (included in GREEN commit)

This deviation is mathematically correct and consistent with the PLAN's `must_haves` truths. The implementation is exactly as specified; only the test scenarios were adjusted to scenarios where the bonus is actually observable.
