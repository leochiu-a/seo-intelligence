# Phase 999.5 Plan 03: EditPopover cluster Autocomplete input + tags prop threading through UrlNode Summary

Cluster tag data entry surface added to EditPopover with multi-value chips, autocomplete suggestions fed by collectClusterSuggestions, and 5-arg onSave threading through UrlNode into data.onUpdate.

## What Was Built

### EditPopover prop surface (before/after)

**Before:**
```typescript
interface EditPopoverProps {
  // ... existing props ...
  onSave: (urlTemplate: string, pageCount: number, isGlobal: boolean, placements: Placement[]) => void;
}
```

**After:**
```typescript
interface EditPopoverProps {
  // ... existing props ...
  tags: string[];                  // NEW — initial tag values from node data
  clusterSuggestions: string[];    // NEW — unique tags from other nodes
  onSave: (
    urlTemplate: string,
    pageCount: number,
    isGlobal: boolean,
    placements: Placement[],
    tags: string[],                // NEW — 5th arg
  ) => void;
}
```

### Tag draft submission flow

Tags are added via two paths:
1. **Plain input (no suggestions):** `value` + `onChange` bound to `tagDraft`. On `Enter` keydown → `e.preventDefault()` + `e.stopPropagation()` → `addTag()` which trims, deduplicates, and appends to `localTags`.
2. **Autocomplete (suggestions present):** Base UI `Autocomplete.Root` with `value={tagDraft}` and `onValueChange`. Enter keydown on `Autocomplete.Input` → same `addTag()`. Selecting an item from the dropdown fires `onValueChange` and sets the draft, user still needs Enter to confirm, or they can press Enter while typing to add a freeform value.

Chips render as flex-wrap with `getClusterColor(tag).dot` colored dot and X remove button. Removing a chip calls `setLocalTags(prev => prev.filter(...))`.

On save (Confirm button, click-outside, or Enter at document level via handleConfirm):
```typescript
const cleanTags = localTags.map((t) => t.trim()).filter((t) => t.length > 0);
onSave(localTemplate.trim(), Math.max(1, localCount), localIsGlobal, localPlacements, cleanTags);
```

### Base UI quirks discovered

- `Autocomplete.Root` requires `onItemHighlighted` prop to avoid a runtime warning when used without that callback — added `onItemHighlighted={() => {}}` noop.
- `Autocomplete.Input` `onValueChange` fires on every keystroke via the root, so `tagDraft` stays in sync. The combobox does not call `onSave` directly — the user must press Enter or click Confirm.
- The `container={popoverRef}` pattern (from Phase 6) correctly keeps the portal inside the popover's DOM subtree so the mousedown-outside handler doesn't mis-fire when the dropdown is open.

### UrlNode changes

- `collectClusterSuggestions` imported alongside `collectPlacementSuggestions`
- `clusterSuggestions` computed via `useMemo([id, getNodes])` — same shape as `placementSuggestions`
- `handleSave` signature extended with `tags: string[]` as 5th param; passed into `data.onUpdate(id, { urlTemplate, pageCount, isGlobal, placements, tags })`
- `tags={data.tags ?? []}` and `clusterSuggestions={clusterSuggestions}` added to EditPopover JSX
