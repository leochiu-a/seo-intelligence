## Cluster Data Model

### Q1: How many clusters can a single node belong to?

| Option | Description | Selected |
|--------|-------------|----------|
| Single cluster | Exactly one (or none). Matches Google's primary topical signal; simpler UI/math. | |
| Multiple tags | 2+ tags possible; requires bonus-compounding rule. | ✓ |

**User's choice:** Multiple tags.

### Q2: How does the PM enter a cluster name?

| Option | Description | Selected |
|--------|-------------|----------|
| Free-text + autocomplete | Phase 6 placement pattern — text input with suggestions from other nodes. | ✓ |
| Fixed preset list | Dropdown of predefined clusters at scenario level. | |
| Color picker only | Pick a color swatch; no name. | |

**User's choice:** Free-text + autocomplete.

### Q3: Is cluster a required field?

| Option | Description | Selected |
|--------|-------------|----------|
| Optional | Unassigned = no cluster (neutral, no bonus). Progressive adoption. | ✓ |
| Required | Every new node must pick. Forces discipline; breaks existing scenarios. | |

**User's choice:** Optional.

### Q4: Do global nodes participate in clusters?

| Option | Description | Selected |
|--------|-------------|----------|
| Yes — full participation | Globals can be tagged; bonus applies to their synthetic edges. | ✓ |
| Display-only | Globals carry tag for color/filter, but bonus NOT applied to synthetic edges. | |
| No — globals skip clusters | Globals ignore cluster entirely. | |

**User's choice:** Yes — full participation.

### Q5 (follow-up): When source and target share MULTIPLE tags, how is bonus computed?

| Option | Description | Selected |
|--------|-------------|----------|
| Any overlap → single bonus | ≥1 shared tag → apply bonus once. Simple, predictable. | ✓ |
| Bonus scales per shared tag | Each shared tag adds a fraction. | |
| Jaccard similarity weighting | Bonus scales by shared/union. | |

**User's choice:** Any overlap → single bonus.

## Weight Math

### Q1: How does same-cluster bonus enter PageRank?

| Option | Description | Selected |
|--------|-------------|----------|
| Multiplicative on effective linkCount | `effectiveLC = linkCount × BONUS`. Cleanly extends existing math. | ✓ |
| Additive post-score boost | Flat bump after normal PageRank. Distorts score semantics. | |
| Cross-cluster dampening | Same-cluster normal; cross-cluster dampened. Penalty framing. | |

**User's choice:** Multiplicative on effective linkCount.

### Q2: Bonus value?

| Option | Description | Selected |
|--------|-------------|----------|
| 1.5× | 50% bump; visible delta without drowning out topology. | ✓ |
| 2× | Double weight; more aggressive. | |
| 1.25× | Conservative 25% bump. | |

**User's choice:** 1.5×.

### Q3: How is the bonus value exposed?

| Option | Description | Selected |
|--------|-------------|----------|
| Module-private const | `CLUSTER_BONUS_FACTOR = 1.5` in graph-utils.ts. Matches Phase 8/10. | ✓ |
| Per-scenario setting | Slider/input per scenario. | |
| Global UI knob | App-level settings panel. | |

**User's choice:** Module-private const.

### Q4: Do same-cluster edges also count for Phase 10 outbound-link math?

| Option | Description | Selected |
|--------|-------------|----------|
| Use RAW linkCount | Outbound counts physical links; bonus is a PageRank trick. | ✓ |
| Apply bonus to outbound count | Internally consistent math; changes Phase 10 semantics. | |

**User's choice:** Use RAW linkCount — ignore cluster bonus.

## Cross-Cluster & Unassigned Behavior

### Q1: How are unassigned nodes treated in the bonus math?

| Option | Description | Selected |
|--------|-------------|----------|
| No overlap possible → no bonus | Empty tag set means empty intersection; no bonus anywhere. | ✓ |
| Implicit 'untagged' cluster | Tag-less nodes share an `__untagged` bucket. Rewards not tagging. | |

**User's choice:** No overlap possible → no bonus.

### Q2: Should cluster info appear in the score sidebar?

| Option | Description | Selected |

**User's choice:** JSON yes, CSV no.
