    **Step 3: Implement `src/hooks/useScenarios.ts`:**

    Export `loadOrMigrate` as a named function (for direct testing).

    The hook `useScenarios` uses `useState<ScenariosStore>` initialized via `loadOrMigrate()`.

    Methods:
    - `createScenario(mode: 'blank' | 'clone', currentNodes, currentEdges)`:
      1. Serialize currentNodes/currentEdges using the same stripping logic as `serializeGraph` in App.tsx (import a shared serializer or inline the strip). Store serialized data into the currently-active scenario slot.
      2. Generate new ID via `crypto.randomUUID()`.
      3. Name: `Scenario ${store.scenarios.length + 1}` (per D-05).
      4. If mode === 'clone': `structuredClone(serializedCurrent)` for nodes/edges. If mode === 'blank': empty arrays.
      5. Add new scenario to store.scenarios, set activeScenarioId to new ID.
      6. Return the new ScenarioRecord.

    - `switchScenario(targetId, currentNodes, currentEdges)`:
      1. Serialize current graph into active scenario slot.
      2. Set activeScenarioId = targetId.
      3. Return target scenario record (caller uses this to call setNodes/setEdges).

    - `renameScenario(id, name)`: Map over scenarios, update matching ID's name.

    - `deleteScenario(id)`:
      1. If scenarios.length <= 1, return null (D-03).
      2. Filter out the scenario.
      3. If deleted was active, set activeScenarioId to first remaining.
      4. Return the new active scenario (or null if deletion was no-op).

    - `persist()`: `localStorage.setItem(SCENARIOS_KEY, JSON.stringify(store))`.

    - `updateActiveGraph(serializedNodes, serializedEdges)`: Updates the active scenario's nodes/edges in the store without changing activeScenarioId. Used by the save effect in App.tsx.

    Note: The hook does NOT call setNodes/setEdges — that is App.tsx's responsibility. The hook only manages the ScenariosStore state and localStorage. This keeps the hook pure-ish and testable without ReactFlow context.
