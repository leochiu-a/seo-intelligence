# Phase 999.5 Plan 05: Same-cluster Edge Coloring in LinkCountEdge Summary

Same-cluster edges rendered with muted cluster hex color via `useReactFlow().getNode` + `getClusterColor().edge`, with `computeEdgeStroke` pure helper extracted for isolated unit testing.

## What Was Built

### Edge Color Precedence Chain

Three-tier priority (highest to lowest):

1. **Selected indigo** (`#6366F1`) — user has selected the edge; always wins
2. **Cluster color** — source and target share ≥1 tag; `getClusterColor(firstSharedTag).edge` hex
3. **Default gray** (`#9CA3AF`) — cross-cluster or untagged endpoints

This matches the existing `selected ? '#6366F1' : '#9CA3AF'` pattern, extended to `selected ? '#6366F1' : (clusterStrokeHex ?? '#9CA3AF')`.

### Deterministic Tie-Break Rule

When source and target share 2+ tags, the implementation:
1. Filters `targetTags` to those present in `sourceSet` (Set lookup, O(1) per tag)
2. Calls `shared.sort()` — in-place lexicographic sort on the filtered array
3. Picks `shared[0]` — first in alphabetical order

Example: source has `['taipei','food']`, target has `['food','taipei']` → shared filtered = `['food','taipei']` → sorted = `['food','taipei']` → picks `'food'` → `getClusterColor('food').edge`.

### How Stroke Is Tested

**Approach used: pure helper testing** (not DOM attribute or inline style).

The `computeEdgeStroke(sourceTags, targetTags, selected)` function is exported from `LinkCountEdge.tsx` and tested directly in `LinkCountEdge.test.tsx`. This was chosen because `EdgeLabelRenderer` (used inside `LinkCountEdge`) calls into React Flow's zustand store, causing tests to error with "Seems like you have not used zustand provider as an ancestor" when rendered without a `ReactFlowProvider`. The pure helper contains the exact same logic as the `clusterStrokeHex` memo + BaseEdge style expression, giving identical coverage.

**For future test maintainers:** If you need to test the full component render (edge path shape, label position), you'll need to wrap with `ReactFlowProvider` or mock the full zustand store. The stroke color branch is fully covered by the helper tests.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Switched to pure helper test pattern due to EdgeLabelRenderer provider requirement**

- **Found during:** RED phase — all 5 test cases errored immediately with React Flow provider error
- **Issue:** `EdgeLabelRenderer` uses `useStore` internally which requires the React Flow zustand `ReactFlowProvider` to be present in the tree. The plan's test strategy used direct component rendering without a provider wrapper.
- **Fix:** Extracted `computeEdgeStroke` as an exported pure function from `LinkCountEdge.tsx` and rewrote the tests to call the helper directly. Added an extra 2 test cases (empty tags branch, no shared tags branch) for better coverage — 7 tests total instead of 5.
- **Plan guidance honored:** The plan explicitly mentioned this fallback: "As a fallback, extract the stroke decision into a pure helper `computeEdgeStroke(sourceTags, targetTags, selected)` exported from the same file and unit-test that helper directly."
