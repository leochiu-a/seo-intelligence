<objective>
Render cluster tags on the UrlNode card: (1) a 4px-wide left-edge color stripe split into up to 3 vertical bands (one per tag), and (2) tag chips in the subtitle row (one chip per tag, text + color dot prefix). Multi-tag overflow (`tags.length > 3`) renders the first 3 stripe bands + a `+N` indicator inline in the subtitle. Stripe-under-dim persistence (D-13) is DEFERRED to Plan 07.

Purpose: Makes cluster membership immediately scannable on the canvas. The stripe is orthogonal to the existing scoreTier border (card-border color from TONE_MAP), so no visual conflict.

Output: UrlNode card visually reflects cluster assignment. No data-flow changes in this plan — only rendering. Reads `data.tags` which was wired in Plan 03.
</objective>

<must_haves>
- Left-edge stripe DIV: `absolute left-0 top-0 h-full w-1 overflow-hidden rounded-l-xl flex flex-col` (or equivalent 4px wide = `w-1`)
- Only rendered when `data.tags && data.tags.length > 0`
- Multi-band: split into equal vertical bands, max 3. Extra tags become the `+N` chip.
- Stripe dim-persistence is DEFERRED to Plan 07 via a `TODO Plan 07` comment in UrlNode.tsx. Plan 04 renders the stripe normally under current `node.style.opacity`; Plan 07's App.tsx + UrlNode refactor closes the gap and removes the TODO.
- Subtitle chips: render AT THE END of the subtitle `<div>` (after over-linked indicator), one `<span>` per tag with `·` separator before the first chip. Each chip: `inline-flex items-center gap-1 text-[11px]` + dot + tag name text.
- `+N` indicator: when `tags.length > 3`, render an extra text span `· +{tags.length - 3}` after the last stripe band (NOT inline chip — just text label).
- No changes to data flow, props interface, or TONE_MAP

Note on depends_on: This plan also depends on Plan 03 (in addition to Plan 02) because it reads `data.tags`, which is wired through `handleSave` / `data.onUpdate` by Plan 03. Both Plan 03 and Plan 04 modify `packages/web/src/components/UrlNode.tsx`, so they must run sequentially (not parallel within Wave 2).
</must_haves>

**Step 3 — Render the left-edge stripe** as the FIRST child inside the outer card `<div>` (immediately after the outer `<div ... className=...>`). Insert:

```tsx
{visibleBandTags.length > 0 && (
  <div
    data-testid="cluster-stripe"
    className="absolute left-0 top-0 bottom-0 w-1 flex flex-col overflow-hidden rounded-l-xl pointer-events-none"
    aria-hidden
  >
    {visibleBandTags.map((tag) => {
      const color = getClusterColor(tag);
      return (
        <div
          key={tag}
          className={`flex-1 ${color.stripe}`}
          data-cluster-tag={tag}
        />
      );
    })}
  </div>
)}
```

The `flex flex-col` + `flex-1` pattern gives equal-height bands regardless of count (1=full, 2=halves, 3=thirds).
