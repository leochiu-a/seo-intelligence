    ```ts
    interface ResizerProps {
      /** Which edge the handle sits on. Determines direction math. */
      side: 'left' | 'right';
      /** Called on every mousemove during drag, with the pixel delta from drag start.
       *  For side='left': delta > 0 means pointer moved left (panel should grow).
       *  For side='right': delta > 0 means pointer moved right (panel should grow).
       *  Consumer is responsible for applying delta to base width and clamping. */
      onResize: (deltaPx: number) => void;
      /** Optional custom test id. Defaults to 'resize-handle' so existing ScoreSidebar tests keep working. */
      'data-testid'?: string;
      /** Optional extra className merged with defaults. */
      className?: string;
    }
    ```

    Tests to write FIRST (in Resizer.test.tsx):
    - Renders a div with role-agnostic `data-testid="resize-handle"` by default, class includes `cursor-col-resize`.
    - Custom `data-testid` prop overrides the default.
    - Mousedown on handle + mousemove on document fires `onResize` with correct delta (side='left': startX - currentX; side='right': currentX - startX).
    - Mouseup stops the drag — subsequent mousemove does NOT call `onResize`.
    - **Regression for the stale-listener bug:** After a full mousedown → mousemove → mouseup cycle, a SECOND mousedown → mousemove DOES fire `onResize` again. (This is the bug we're fixing — the old ScoreSidebar's effect-mounted listeners failed this.)
    - Unmounting during an active drag removes document `mousemove` and `mouseup` listeners (use `vi.spyOn(document, 'removeEventListener')`).
    - Mousedown calls `e.preventDefault()` (verify via `preventDefault` spy on the event).

    Implementation approach (write AFTER tests fail):
    - No `useEffect` with `[]` deps. Instead, `handleMouseDown` (called from React onMouseDown) attaches fresh `mousemove` + `mouseup` listeners via `document.addEventListener`. `mouseup` handler detaches BOTH listeners. This gives each drag its own listener lifecycle → fixes the stale-listener bug.
    - Track drag start via `useRef<number | null>(null)` for `startX`. Reset to null on mouseup.
    - For unmount cleanup: a `useEffect(() => () => {...}, [])` cleanup that detaches any listeners still attached (use a ref holding the current handler pair so cleanup can find them).
    - Render: `<div data-testid={testId} onMouseDown={handleMouseDown} className={cn('cursor-col-resize hover:bg-blue-400/40 select-none', className)} />`. No absolute positioning inside the component — positioning is the consumer's responsibility (keep it layout-agnostic for reuse).

    Per global rule: commit message `feat(resizer): add reusable Resizer component with per-drag listener lifecycle` after tests pass.
