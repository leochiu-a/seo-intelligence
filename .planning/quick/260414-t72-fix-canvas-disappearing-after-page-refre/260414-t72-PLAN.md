<objective>
Fix bug: canvas disappears after page refresh when JSON has been imported.

Root cause: The `isRestoring` ref guard in the save effect (line 319) fails because React effects execute in definition order within the same commit — the restore effect sets `isRestoring.current = false` BEFORE the save effect runs, but the save effect still sees the initial empty `nodes`/`edges` state (setState is async). This overwrites localStorage with `{nodes:[], edges:[]}`. In StrictMode (double-invocation), the second restore reads the now-empty localStorage and finds nothing.

Fix: Replace `isRestoring` with `isFirstRender` — the save effect unconditionally skips its first invocation (regardless of restore timing), then sets the flag to false. Subsequent renders (including the re-render triggered by restore's setNodes/setEdges) proceed normally and save correct data.

Purpose: Critical data-loss bug — users lose imported work on every refresh.
Output: Regression test + 3-line fix in App.tsx.
</objective>
