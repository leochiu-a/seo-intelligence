<objective>
Fix the bug where /zh-tw (or any node that is both `isRoot=true` and `isGlobal=true`) receives a "LOW" score tier despite being the designated root of the graph.

Purpose: The root/homepage is semantically the most authoritative entry point in a site structure. It should receive a PageRank boost reflecting that all crawls originate there. Currently `isRoot` is completely ignored by `calculatePageRank` — only `isGlobal` nodes get synthetic inbound injection, and only when they have `placements` with `linkCount > 0`. A node that is both root AND global but lacks placements (or zero-link placements) gets no boost at all.

Root cause (traced in graph-pagerank.ts lines 85–106):
- Synthetic inbound injection fires only for `isGlobal` nodes with `totalPlacementLinks > 0`
- If `/zh-tw` is global but has no placements, the `continue` guard skips it
- `isRoot` flag is entirely absent from calculatePageRank — only used for BFS in calculateCrawlDepth
- classifyScoreTier is percentile-based (bottom 1/3 → low), so if /zh-tw gets near-zero synthetic boost it lands in "low"

Fix: inject synthetic inbound into the root node from all non-root nodes, regardless of whether the root is also global. Use a fixed synthetic linkCount of 1 (similar to the global injection default) so the root node always has meaningful inbound rank transfer. This mirrors how PageRank models the real web: the root is linked from everywhere.

Output: Updated calculatePageRank accepting optional rootId, new test cases confirming root node escapes "low" tier.
</objective>

```
// Root node injection: every non-root node implicitly links to the root node.
// Rationale: the root/homepage is the canonical entry point — all crawls originate
// there. This mirrors the global-node injection pattern but uses a fixed linkCount=1
// (no placement-count weighting). Skipped when rootId is null/undefined or when
// the root is already a global node (to avoid double-counting if it also has placements).
if (rootId) {
  const rootNode = nodes.find((n) => n.id === rootId);
  if (rootNode && !rootNode.data.isGlobal) {
    const syntheticLinkCount = 1;
    for (const sourceNode of nodes) {
      if (sourceNode.id === rootId) continue; // skip self-link
      const sameCluster = hasSameCluster(sourceNode.data.tags, rootNode.data.tags);
      inbound.get(rootId)?.push({
        sourceId: sourceNode.id,
        linkCount: syntheticLinkCount,
        sameCluster,
      });
      outbound.get(sourceNode.id)?.push({
        targetId: rootId,
        linkCount: syntheticLinkCount,
        sameCluster,
      });
    }
  }
}
```

Note the guard `!rootNode.data.isGlobal`: if the root IS also a global node it already receives synthetic inbound from the global injection (provided it has placements). However if the root is global but has zero placement links it still gets skipped by the global `continue` guard — so the root injection should also fire for `isGlobal` roots with no effective placements. Adjust guard:

```typescript
const rootAlreadyGlobalInjected =
  rootNode.data.isGlobal &&
  (rootNode.data.placements ?? []).reduce((s, p) => s + p.linkCount, 0) > 0;

if (!rootAlreadyGlobalInjected) {
  // inject synthetic inbound from all non-root nodes
}
```

The `totalWeightedOut` precompute block runs AFTER both injection blocks, so the new synthetic outbound edges from non-root nodes into the root are included in the denominator — consistent with how global injection is handled.
